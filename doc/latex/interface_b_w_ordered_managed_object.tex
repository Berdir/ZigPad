\subsection{Detailed Description}
A subclass of NSManagedObject that provide support for imposing an ordering on to-\/many relationships

By default, to-\/many Core Data relationships are unordered, but there are some cases where you want to impose a specific ordering on a relationship. The classic example of this is iTunes, where each playlist can have a list of tracks, but those tracks can be put in a specific order by the user.

\hyperlink{interface_b_w_ordered_managed_object}{BWOrderedManagedObject} manages this by using an additional property to store the ordering for each relationship that you want to be ordered. By default, the string \char`\"{}Ordering\char`\"{} is appended to the name of the relationship key to form the key that the ordering data is stored in (e.g. the ordering for the \char`\"{}tracks\char`\"{} relationship is stored under the key \char`\"{}tracksOrdering\char`\"{})

\hyperlink{interface_b_w_ordered_managed_object}{BWOrderedManagedObject} also provides a set of methods that allow for inserting, deleting, and rearranging objects in a specific order.
